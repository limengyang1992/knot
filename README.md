# KNOT — Knowledge eNtanglement unlearning bencOmark Toolkit

KNOT is an automatic pipeline for constructing machine unlearning benchmark datasets.
It generates structured QA pairs across three knowledge types (**entity**, **concept**, **skill**),
labels each pair with an **entanglement score** (how tightly "forget" knowledge is coupled
with "retain" knowledge), and organises everything into three splits ready for unlearning evaluation.

---

## What You Will Get

After running the pipeline you will find all outputs under `knot_data/`:

```
knot_data/
├── dataset_stats.json          # Summary counts
├── raw/                        # Intermediate entity triples
├── knot_entity/
│   ├── forget_final.jsonl      # Private/personal facts to be forgotten
│   ├── retain_final.jsonl      # Public achievements to be retained
│   └── boundary_final.jsonl    # Questions that need both types of knowledge
├── knot_concept/               # Same structure for concept tasks
└── knot_skill/                 # Same structure for skill tasks
```

Each `.jsonl` record looks like:

```json
{
  "question": "What is Albert Einstein's place of birth?",
  "answer": "Ulm, Germany",
  "triple_ref": "place of birth",
  "entity_id": "Q1000",
  "entity_name": "Albert Einstein",
  "split": "forget",
  "entanglement_score": 0.024,
  "entanglement_level": "High"
}
```

**Entanglement levels** — `Low / Medium / High / Extreme` — are assigned by quartile,
measuring how closely the forget fact is coupled to retain facts via embedding similarity,
KG distance, and LLM-judge scores.

Default scale (configurable):

| Task | Forget | Retain | Boundary |
|------|-------:|-------:|---------:|
| Entity | ~600 | ~1500 | ~300 |
| Concept | ~100 | ~140 | ~100 |
| Skill | ~60 | ~60 | ~20 |

## Requirements

- Python 3.9+
- A [DeepSeek API key](https://platform.deepseek.com)

## Setup

```bash
# 1. Clone
git clone https://github.com/limengyang1992/knot.git
cd knot

# 2. Install dependencies
pip install openai jsonlines sentence-transformers numpy

# 3. Set your API key (never hard-code it)
export DEEPSEEK_API_KEY=sk-your_key_here
```

---

## Running the Pipeline

```bash
bash run_all.sh
```

The script runs **6 phases** in order:

| Phase | What it does |
|-------|-------------|
| 1 | Generate ~100 globally known entities (scientists, politicians, artists, athletes…) with knowledge triples via DeepSeek, then produce Forget/Retain/Boundary QA pairs |
| 2 | Generate concept list and concept-level QA pairs |
| 3 | Generate skill-level QA pairs |
| 4 | Deduplicate all QA pairs; LLM-verify quality |
| 5 | Compute entanglement scores (embedding similarity + KG distance + LLM judge), assign levels |
| 6 | Print and save summary statistics |

### Resuming After a Failure

Each step writes a checkpoint file to `.checkpoints/` when it succeeds.
If the pipeline is interrupted, just re-run `bash run_all.sh` — completed steps are skipped automatically.

```bash
# Re-run a specific step (delete its checkpoint)
rm .checkpoints/phase4_verify.done

# Start completely from scratch
rm -rf .checkpoints knot_data
```


## Notes

- **Entity data** is generated by DeepSeek using globally known real-world figures; it is
  LLM-produced and not fetched live from Wikidata.
- **LLM verification** (Phase 4) uses a fail-open strategy: if the API is unreachable,
  all items are kept rather than discarded.
- **API cost** depends on dataset scale. The default run makes roughly 400–500 API calls.
  Reduce `TARGET_PER_OCC` in `query_wikidata_entities.py` to generate fewer entities and
  lower cost during testing.
